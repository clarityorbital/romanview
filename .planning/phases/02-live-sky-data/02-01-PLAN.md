    Create `src/hooks/useGaiaStars.ts` implementing research Pattern 4 (debounced async query with AbortController).

    Export the hook and its types:
    ```typescript
    export type QueryState = 'idle' | 'loading' | 'loaded' | 'error';

    export interface QueryStatus {
      state: QueryState;
      count?: number;
      magLimit?: number;
      isDense?: boolean;
      message?: string;
    }

    export function useGaiaStars(ra: number | undefined, dec: number | undefined): {
      stars: GaiaSource[];
      status: QueryStatus;
    }
    ```

    Implementation:
    - Import `adaptiveGaiaQuery` and `GaiaSource` from `../lib/vizier`
    - Use `useState` for stars (GaiaSource[]) and status (QueryStatus)
    - Use `useRef` for AbortController
    - Use `useEffect` on [ra, dec]:
      - If ra or dec is undefined, set state='idle', clear stars, return
      - Abort previous request via `abortRef.current?.abort()`
      - Create new AbortController, store in ref
      - Set state='loading'
      - Start 300ms debounce timer via setTimeout
      - On timer fire: call `adaptiveGaiaQuery(ra, dec, signal)`
      - On success (if not aborted): setStars, setStatus with count/magLimit/isDense
      - On error (if not aborted): setStatus state='error' with message
      - Cleanup: clearTimeout + abort controller

    Key behaviors from research anti-patterns:
    - Only fires when RA/Dec changes (not on camera pan/zoom)
    - 300ms debounce prevents rapid-fire queries
    - AbortController prevents stale data from overwriting current results
    - Does NOT re-fetch on position angle changes (stars don't change when PA changes)
