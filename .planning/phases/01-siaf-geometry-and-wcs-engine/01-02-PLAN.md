    Rewrite FocalPlaneView.tsx to use the centralized wcs.ts projection and SIAF-derived detector positions. This task also fixes the critical 60-degree PA error.

    **Fix the PA chain:**
    The current code computes `pa = positionAngle(target, sun)` which gives V3PA. For the focal plane view:
    - The aperture PA (APA) = V3PA - V3IdlYAngle = V3PA - (-60) = V3PA + 60
    - Use V3PA when calling `skyToFocalPlane` (it expects V3PA, not APA, because it works in the V3 frame)
    - BUT: the N/E compass rose and the detector layout in the SVG need to account for APA.

    Actually, since `skyToFocalPlane` rotates sky coordinates by -V3PA to get to the V3 frame, and the detectors in V2V3 frame are already in their physically rotated positions, the focal plane view should show detectors in V2V3 coordinates (not sky coordinates). Here is the correct approach:

    **Detector rendering in focal plane SVG:**
    - Detectors should be drawn using their V2V3 corner positions (relative to boresight), converted to arcminutes.
    - For each detector, compute corner offsets from boresight: `dv2 = corner_v2 - boresight_v2`, `dv3 = corner_v3 - boresight_v3`.
    - Convert to arcminutes: `x = dv2 / 60`, `y = dv3 / 60` (V2V3 are in arcseconds).
    - Note: In the SVG, use x = dv2/60 for horizontal and y = -dv3/60 for vertical (SVG Y is inverted).
    - This shows the focal plane in its "native" orientation -- the physical detector layout.

    **Star projection in focal plane:**
    - Use `skyToFocalPlane(starRa, starDec, targetRa, targetDec, v3pa)` to get star positions in arcminutes.
    - `skyToFocalPlane` returns positions in the focal plane frame (rotated by -V3PA from sky).
    - These positions are in the tangent plane rotated to align with V2/V3, so they should align with the detector positions drawn above.
    - NOTE: skyToFocalPlane returns {x, y} in arcminutes where x corresponds to the V2 direction and y corresponds to V3. Apply the same V2 sign convention as v2v3ToSky: skyToFocalPlane should return x = -xi_rotated (in arcmin) and y = eta_rotated (in arcmin) to match V2V3 convention. Verify this is handled in wcs.ts; if not, apply the sign flip here.

    **Star pre-filtering:**
    Fix the high-declination bug (Pitfall 5 from RESEARCH.md):
    - Replace `if (dRa > 1 && dRa < 359) continue;` with proper cos(dec) correction:
      ```typescript
      const searchRadius = 1.0; // degrees
      const cosDec = Math.cos(targetDec * Math.PI / 180);
      const dRaThreshold = cosDec > 0.01 ? searchRadius / cosDec : 360;
      if (dRa > dRaThreshold && dRa < (360 - dRaThreshold)) continue;
      ```

    **N/E compass rose:**
    The compass should show where North and East are in the focal plane frame. Since the focal plane is rotated by V3PA from the sky:
    - North in focal plane: rotate (0, 1) by -V3PA
    - East in focal plane: rotate (1, 0) by -V3PA
    - Use `rotateByPA` from wcs.ts for this.

    **SVG viewBox:**
    Compute FOV bounds dynamically from SIAF detector corners instead of using the old TOTAL_FOV_ARCMIN constants:
    ```typescript
    const allCorners = WFI_DETECTORS.flatMap(d =>
      d.corners_v2v3.map(([v2, v3]) => ({
        x: (v2 - WFI_BORESIGHT.v2) / 60,
        y: (v3 - WFI_BORESIGHT.v3) / 60
      }))
    );
    const minX = Math.min(...allCorners.map(c => c.x));
    const maxX = Math.max(...allCorners.map(c => c.x));
    const minY = Math.min(...allCorners.map(c => c.y));
    const maxY = Math.max(...allCorners.map(c => c.y));
    const padFactor = 1.15;
    ```
    Use these bounds for the SVG viewBox.

    **Detector labels:** Use `det.id` directly (now "WFI01" etc.). Remove the `.replace('SCA', '')` call -- instead show just the number portion: `det.id.replace('WFI', '')`.

    **PA display in header:** Show APA (V3PA + 60) as the displayed "PA" since that is the aperture orientation on the sky.

    Keep the same component props interface -- no App.tsx changes needed.
