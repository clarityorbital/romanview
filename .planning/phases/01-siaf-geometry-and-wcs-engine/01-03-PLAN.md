    Human visual verification required. Steps:

    1. Start the dev server: `cd /workspace/RomanView && npm run dev`
    2. Open the app in a browser (typically http://localhost:5173)

    3. Select any target (e.g., add a target at RA=180, Dec=0 or search for a known object)
    4. Verify the WFI footprint on the celestial sphere shows a distinctive "3 rows x 6 columns" layout
    5. The layout should be visibly rotated ~60 degrees from a simple horizontal grid
    6. There should be visible gaps between detectors (non-uniform -- wider between columns, narrower between rows)
    7. Compare against the official WFI focal plane diagram: https://roman.gsfc.nasa.gov/science/WFI_technical.html

    **Detector labels (FOOT-02):**
    8. Zoom in to read detector labels on the sky view
    9. Labels should read WFI01 through WFI18 (NOT SCA01-SCA18)
    10. The layout should be column-major:
        - Bottom-left region: WFI07, WFI08, WFI09 (column 1)
        - Center-left: WFI04, WFI05, WFI06 (column 2)
        - Center: WFI01, WFI02, WFI03 (column 3)
        - Center-right: WFI10, WFI11, WFI12 (column 4)
        - Right: WFI13, WFI14, WFI15 (column 5)
        - Far-right: WFI16, WFI17, WFI18 (column 6)

    **Position angle rotation (FOOT-03):**
    11. Change the epoch using the epoch slider
    12. The entire footprint should rotate around the boresight as the epoch changes
    13. The rotation should be in the East-of-North direction (counterclockwise when North is up)
    14. The focal plane view's N/E compass should update to reflect the new PA

    **Focal plane view (FOOT-04):**
    15. Check the focal plane panel (top-left corner)
    16. Detectors should appear as rectangles in the same layout as the sky view
    17. The PA value displayed should be ~60 degrees different from what you'd expect from just the Sun angle
    18. If there are bright stars in the field, they should appear as dots at plausible positions on the detectors

    **Regressions:**
    19. Verify no other UI elements are broken (target list, epoch slider, observability timeline)
    20. Verify the app does not crash or show console errors
  </verify>
